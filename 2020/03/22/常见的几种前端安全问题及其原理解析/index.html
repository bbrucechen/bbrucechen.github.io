Web安全

常见的几种前端安全问题及其原理解析

Posted by Bruce Chen on 2020-03-22

虽然在很多人眼里前端工程师就是写写页面写写交互逻辑，然后调调接口把数据渲染上去就完事了，至于网络安全什么的问题都跟这个职位完全无关。但是其实作为每个互联网用户最直接接触到的前端页面来说，前端往往是网络安全发生漏洞的重灾区（尽管大部分漏洞都需要后端来修复），下面就说几个常见web前端安全问题：

  1. CSRF（cross site request forgery：跨站请求伪造）
  2. XSS（cross site script：跨站攻击脚本）
  3. SQL注入

# 1.CSRF

CSRF就是跨站请求伪造，即攻击者利用用户的cookie将自己伪造成合法用户从而对目标网站发起恶意请求。我们通过一张图来看下其基本原理

CSRF攻击需要满足以下几个条件：
1.目标网站A使用cookie来验证用户身份
2.用户登录过目标网站A（即cookie存在硬盘中）
3.用户没有登出过目标网站A

满足了以上几个条件，我们再来具体看看CSRF的具体过程：

首先是用户登录了目标网站A（步骤1），网站A的服务器发送了Cookie给用户，使得用户身份被验证（步骤2）。然后用户不小心访问了攻击者的恶意网站B（步骤3），此时在网站B上攻击者通过某种手段使得用户在不知情的情况下带着原先的cookie发起了对目标网站A的恶意请求（步骤4），网站A接收到之后检验cookie误以为是用户发起的请求并做出回应，从而使攻击者达到目的

可以看出，整个CSRF其实利用的都是用户的自发行为，所以前端基本没有什么办法防御，只能靠后端对请求进行甄别：
1.通过检验请求的referer字段：referer字段表示的是用户通过哪个网站对网站发起请求，但是这个方法依赖的是浏览器能够正确发送referer字段，如果是通过代理那么这个方法则不可靠了
2.通过在请求地址中加token并验证
3.用户提高自身安全意识，避免访问恶意网站
